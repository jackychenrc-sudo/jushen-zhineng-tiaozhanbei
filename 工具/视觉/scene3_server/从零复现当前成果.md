# 从零复现闫帅辰当前 Scene3 成果

## 1. 这份文档能复现什么

这份文档只复现当前已经在比赛仿真服务器上验证过的阶段性成果：

1. 读取 Scene3 头部 RGB-D 相机。
2. 识别货架上层 3 个随机摆放的 SMT 料盘。
3. 输出像素框、深度、相机坐标和 base_link 坐标。
4. 连续采集 3 至 5 帧，验证目标数量、位置和深度是否稳定。
5. 根据稳定视觉结果生成任务规划干跑结果。
6. 调用官方 IK/FK 服务做站距可达性分析，但不移动机械臂。
7. 在人工确认后使用 V4 闭环程序执行一次受限向前短步，停下并重新识别。

当前不能复现的内容：

- 机械臂真实预抓取。
- 夹爪闭合与料盘夹取。
- 抽出、转移和放入出库区。
- 下层料盘任务。

因此，本流程不是完整比赛程序，也不能直接获得 Scene3 上层 10 分。

## 2. 安全与比赛规则

整个复现过程不得读取或调用：

- /mujoco/qpos
- /ground_truth/state
- /set_object_position
- /set_object_position_lock

不得修改评分逻辑、受保护初始化模块和场景模型，也不能根据固定 seed 写死目标位置。

各程序的动作边界：

| 程序 | 是否会移动机器人 |
|---|---|
| scene3_environment_probe.py | 否 |
| scene3_dataset_collector.py | 否 |
| scene3_upper_tray_perception.py | 否 |
| scene3_multiframe_stability.py | 否 |
| scene3_upper_task_planner.py | 否 |
| scene3_upper_reachability.py | 否，只调用 IK/FK 计算 |
| scene3_upper_approach.py 普通运行 | 否，只输出计划 |
| scene3_upper_approach.py 带 --execute | 旧开环接口，不作为本次复现方案 |
| scene3_closed_loop_velocity_motion.py 普通运行 | 否，只输出 V4 移动计划 |
| scene3_closed_loop_velocity_motion.py 带 --execute | 是，只执行一次受限底盘短步，不控制手臂和夹爪 |

第一次复现时，先完成所有只读步骤。没有看到 3 个正确绿色框、没有通过多帧验证时，禁止使用 --execute。

### 8.2 框选一个料盘

Docker 必须具有可用的 DISPLAY。只框一个完整的上层料盘，尽量贴紧料盘竖直主体，不要把两个料盘、货架横杆或大片背景一起框进去。

~~~bash
RAW=/root/kuavo_ws/vision_dataset/scene3_template/seed_005/head/template_source/rgb/seed_005_head_frame_000.jpg
TEMPLATE=/root/kuavo_ws/vision_debug/templates/upper_tray_template.png

mkdir -p /root/kuavo_ws/vision_debug/templates

RAW="$RAW" TEMPLATE="$TEMPLATE" python3 - <<'PY'
import json
import os
from pathlib import Path

import cv2

raw_path = Path(os.environ["RAW"])
template_path = Path(os.environ["TEMPLATE"])

image = cv2.imread(str(raw_path))
if image is None:
    raise SystemExit("无法读取图片: {}".format(raw_path))

x, y, w, h = cv2.selectROI(
    "Select ONE complete upper tray, then press ENTER",
    image,
    showCrosshair=True,
    fromCenter=False,
)
cv2.destroyAllWindows()

if w <= 0 or h <= 0:
    raise SystemExit("没有选择有效区域")

template = image[y:y + h, x:x + w]
template_path.parent.mkdir(parents=True, exist_ok=True)

if not cv2.imwrite(str(template_path), template):
    raise SystemExit("模板保存失败")

metadata = {
    "source_image": str(raw_path),
    "bbox_xyxy": [int(x), int(y), int(x + w), int(y + h)],
    "template_size": [int(w), int(h)],
    "usage": "appearance template only; never use bbox as a fixed competition position",
}
template_path.with_suffix(".json").write_text(
    json.dumps(metadata, ensure_ascii=False, indent=2),
    encoding="utf-8",
)

print("模板保存成功 =", template_path)
print("bbox =", metadata["bbox_xyxy"])
print("template_size =", metadata["template_size"])
PY
~~~

当前已验证过的模板大约是细长竖直框，但不同相机距离下尺寸可以不同。算法会做多尺度匹配，不能因此把宽高或位置写死。

## 13. 可选：复现一次 V4 真实闭环短步

执行本节前必须确认：

- 仿真窗口正常运行。
- CURRENT 指向刚生成的多帧稳定 JSON。
- 3 个绿色框全部正确。
- `/humanoid_get_current_gait_name` 返回 `stance`。
- 使用刚启动的干净 Scene3，机器人前方没有障碍。
- 周围没有其他队员同时控制机器人。

先做 V4 干跑。下面命令不会移动机器人：

~~~bash
MOTION="$BASE/scene3_closed_loop_velocity_motion.py"
PULSE=/root/kuavo_ws/vision_debug/reproduce_pulse_1
mkdir -p "$PULSE"

python3 "$MOTION" \
  --forward 0.05 \
  --maximum-lateral-speed 0.01 \
  --stop-margin 0.025 \
  | tee "$PULSE/motion_dry_run.json"
~~~

确认输出同时满足：

- `mode` 为 `dry_run`。
- `controller_version` 为 `foot_midpoint_holonomic_v4`。
- `feedback` 包含 `midpoint(leg_l6_link, leg_r6_link)`。
- `relative_target.forward_m` 为 `0.05`。

再执行一次真实短步。该命令会控制底盘前进，并做有限的横向和偏航纠偏；不会控制机械臂或夹爪：

~~~bash
python3 "$MOTION" \
  --forward 0.05 \
  --maximum-lateral-speed 0.01 \
  --stop-margin 0.025 \
  --execute \
  --confirmation CLOSED_LOOP_VELOCITY \
  --output "$PULSE/result.json" \
  | tee "$PULSE/run.log"

MOVE_RC=${PIPESTATUS[0]}
echo "移动退出码=$MOVE_RC"

if [ "$MOVE_RC" -ne 0 ]; then
  echo "移动程序失败，禁止继续"
fi
~~~

只有 `result.json` 中 `status=completed`、`success=true`，并且四项 `verification_results` 全部为 `true`，才算本次底盘短步通过。无论成功还是失败，都先等待机器人完全站稳：

~~~bash
sleep 5
~~~

然后采集 5 帧，不要立即执行第二次移动：

~~~bash
AFTER=/root/kuavo_ws/vision_debug/reproduce_after_pulse_1
mkdir -p "$AFTER"

python3 "$BASE/scene3_multiframe_stability.py" \
  --capture \
  --capture-count 5 \
  --frame-delay 0.7 \
  --template "$TEMPLATE" \
  --output-dir "$AFTER" \
  > "$AFTER/run.log" 2>&1

VISION_RC=$?
echo "视觉复检退出码=$VISION_RC"
tail -n 12 "$AFTER/run.log"
~~~

先验证结果，再计算距离。下面脚本不会对空列表取中位数：

~~~bash
BEFORE_JSON="$CURRENT" \
AFTER_JSON="$AFTER/stable_upper_trays.json" \
python3 - <<'PY'
import json
import os
import statistics
import sys
from pathlib import Path

before_path = Path(os.environ["BEFORE_JSON"])
after_path = Path(os.environ["AFTER_JSON"])

if not after_path.exists():
    print("移动后没有生成稳定结果，禁止继续前进")
    raise SystemExit(2)

before = json.loads(before_path.read_text())
after = json.loads(after_path.read_text())

def valid(data):
    trays = data.get("upper_trays", [])
    validation = data.get("temporal_validation", {})
    return (
        data.get("status") == "ok"
        and validation.get("passed") is True
        and len(trays) == 3
    )

if not valid(before):
    print("移动前 JSON 无效，禁止继续")
    raise SystemExit(2)

if not valid(after):
    print("移动后视觉无效，禁止继续")
    print("状态 =", after.get("status"))
    print("数量 =", len(after.get("upper_trays", [])))
    print("错误 =", after.get("temporal_validation", {}).get("errors", []))
    raise SystemExit(2)

def distance(data):
    return statistics.median(
        float(tray["base_link_xyz_raw_m"][0])
        for tray in data["upper_trays"]
    )

before_distance = distance(before)
after_distance = distance(after)
movement = before_distance - after_distance

safe = 0.005 <= movement <= 0.12

print("移动前 =", round(before_distance, 3), "m")
print("移动后 =", round(after_distance, 3), "m")
print("视觉位移 =", round(movement, 3), "m")
print("多帧通过 =", after["temporal_validation"]["passed"])
print("数量 =", len(after["upper_trays"]))
print("安全 =", safe)

if not safe:
    print("位移异常，禁止继续前进")
    raise SystemExit(2)
PY
~~~

只有脚本输出安全=True，并且人工检查 5 帧绿色框都正确时，才能手工更新：

~~~bash
CURRENT="$AFTER/stable_upper_trays.json"
~~~

然后重新执行干跑，再决定是否进行下一次单步。当前只验证过一次 V4 `0.05 m` 实机短步，不要把多次移动写成无人看管的连续循环，也不要为了“更容易通过”擅自放宽横漂、偏航或视觉门槛。视觉失败时立即停止，不能根据上一次距离继续移动。

## 16. 常见故障

### Unable to communicate with master

Scene3 没有运行，或者进入了错误容器。回终端 A 检查仿真。

### 找不到 rospy、cv2 或 tf2_ros

没有进入比赛容器，或者没有执行：

~~~bash
source /root/kuavo_ws/devel/setup.bash
~~~

### 找不到模板

重新执行第 8 节，确认：

~~~bash
ls -lh /root/kuavo_ws/vision_debug/templates/upper_tray_template.png
~~~

### status 为 count_mismatch 或 rejected_input

至少一帧没有检测到 3 个可靠料盘。查看对应 frames/frame_XXX/upper_trays.jpg、upper_candidates.jpg 和 perception.log。禁止继续移动。

### no median for empty data

旧检查命令在确认 upper_trays 非空前就计算了中位数。使用本文件第 13 节的新检查脚本。出现该错误后不能沿用旧 CURRENT 继续前进。

### TF_OLD_DATA

先确认仿真没有被重启后沿用旧 ROS 节点。必要时关闭当前任务节点，重新启动一次 Scene3，再重新采集，不能混用重启前后的 JSON。

### GitHub 拉取超时

不要删除服务器上的可运行代码。保留旧目录，稍后重试 git pull，或者在宿主机下载个人分支后使用 docker cp 复制到容器。
