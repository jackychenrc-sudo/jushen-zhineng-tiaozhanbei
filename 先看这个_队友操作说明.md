# 先看这个：队友操作说明

这份文档只说明三件事：每个人用哪个分支、每个文件夹有什么用、文件应该传到哪里。具体怎么实现不限制，大家可以自由发挥。

## 基本原则

```text
1. 不直接改 main。
2. 每个人只在自己的分支上传和修改。
3. 文件不要乱传到仓库根目录。
4. 大文件不上传 GitHub。
5. 每天晚上同步一次：进度、问题、下一步。
```

| 你要传的东西 | 传到哪里 |
| --- | --- |
| Scene1 代码/说明/小结果 | `任务代码参考/场景一_包裹称重入箱/` |
| Scene2 代码/说明/小结果 | `任务代码参考/场景二_零件分拣/` |

```text

2026-07-12_王森_scene3_夹爪测试说明.md
2026-07-12_王仔俊_scene1_包裹截图.jpg
```

## 怎么传文件

会用 Git 的同学按自己的习惯提交到个人分支。

不会用 Git 的同学用 GitHub 网页：

```text
1. 打开仓库页面。
2. 左上角分支选择自己的分支，不要选 main。
3. 进入对应文件夹。
4. 点 Add file / Upload files。
5. 把文件拖进去。
6. 下面 Commit changes，提交到自己的分支。
```

如果只是改 README 或测试记录，也是在自己的分支里点铅笔编辑，然后 Commit changes。

## 上传后必须写清楚

传完文件后，在提交说明、README 或 `文档/测试记录.md` 里写清楚：

```text
这个文件是什么？
对应哪个 scene 和 seed？
跑到了哪一步？
成功了什么？
卡在哪里？
```

## 不要上传到 GitHub 的东西

```text
build/
devel/
log/
*.bag
*.mp4
*.avi
*.mov
*.pt
*.pth
大型数据集
大模型权重
```

大文件放服务器路径或网盘，在 `文档/测试记录.md` 里写清楚路径即可。
