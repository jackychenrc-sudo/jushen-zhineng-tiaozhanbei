# 先看这个：团队协作边界说明

这份文档只说明分工、分支、文件放哪里和交付边界，不规定具体实现方法。大家可以自由发挥，只要每天能留下可验证的结果。

## 基本原则

```text
1. 不直接改 main。
2. 每个人在自己的分支推进。
3. 文件不要乱传到仓库根目录。
4. 大文件不上传 GitHub。
5. 每天晚上同步一次：进度、问题、下一步。
6. 能跑、能解释、能提交，比方案复杂更重要。
```

不要把文件直接丢到仓库根目录。按下面放：

## 怎么传文件

会用 Git 的同学：传到自己的分支，晚上说明改了什么。

不会用 Git 的同学：可以用 GitHub 网页上传，注意两点：

```text
1. 先切到自己的分支，不要直接传 main。
2. 点 Add file / Upload files，把文件拖到对应目录，然后 Commit changes。
```

如果传的是运行成果，上传后在 `文档/测试记录.md` 或当天说明里写清楚：

```text
这个文件是什么？
对应哪个 scene 和 seed？
成功了什么？
卡在哪里？
```

## 每个人每天需要留下什么

每个人每天至少同步这几项：

```text
今天做了什么：
现在能跑到哪一步：
有什么截图/视频/输出文件：
文件放在哪里：
卡在哪里：
明天准备做什么：
```

不要求格式很漂亮，但必须能让别人接上。

## 各组目标

### Scene1：王仔俊

目标是包裹称重与入箱。第一阶段先证明能处理 1 个包裹，再考虑扩展到 4 个。

### Scene2：史智涛 + Kevin

目标是零件识别、分类和放入对应盒子。第一阶段先证明能识别并处理 1 个零件，再扩展到 6 个。

### Scene3：闫帅辰 + 王森

目标是 SMT 料盘出库。第一阶段先处理上层料盘，再尝试下层料盘。

### 视觉与坐标：闫帅辰，王仔俊辅助

目标是给动作模块提供“物体是什么、在哪里、深度/坐标大概是多少”。具体检测方法不限制。

### 测试与提交：窦欣悦

目标是每天记录真实运行结果，最后整理提交包、README、截图视频和测试记录。

## 晚上同步看什么

每天晚上只看这些：

```text
1. 哪些东西今天真的跑过？
2. 哪些结果有截图、视频或输出文件？
3. 文件放在哪里？
4. 哪些问题卡住了别人？
5. 明天每个人最小交付是什么？
```

## 不要上传到 GitHub 的东西

```text
build/
devel/
log/
*.bag
*.mp4
*.avi
*.mov
*.pt
*.pth
大型数据集
大模型权重
```

大文件放服务器路径或网盘，在测试记录里写清楚路径即可。
