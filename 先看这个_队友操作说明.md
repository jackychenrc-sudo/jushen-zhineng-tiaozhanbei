## 4. 每天开工前怎么同步

每个人都在自己的分支里执行：

```bash
git fetch origin
git checkout 自己的分支名
git pull origin 自己的分支名
```

如果公共说明更新了，再合一下中文结构：

```bash
git merge origin/中文结构整理
```

如果出现冲突，不要乱改，先截图发群里。

## 5. 每天写完怎么提交

先看改了哪些文件：

```bash
git status
```

添加文件：

```bash
git add .
```

提交：

```bash
git commit -m "写一句你做了什么"
```

例子：

```bash
git commit -m "vision: 保存depth并输出料盘3D坐标"
git commit -m "scene3: 添加上层料盘固定动作骨架"
git commit -m "scene1: 保存包裹RGB和depth样本"
git commit -m "scene2: 添加零件候选框检测"
git commit -m "docs: 更新测试记录"
```

推到 GitHub：

```bash
git push origin 自己的分支名
```

例子：

```bash
git push origin scene3-wangsen
```

## 6. 每天晚上怎么合并

每天晚上统一做：

```text
1. 每个人 push 自己分支。
2. 到 GitHub 网页开 Pull Request。
3. PR 标题写中文：例如“Scene3：王森添加动作骨架”。
4. 窦欣悦检查有没有测试记录、截图/视频路径。
5. 闫帅辰看能不能合并。
6. 能跑的合进 main；不能跑的继续留在个人分支。
```

## 9. 闫帅辰：视觉与坐标怎么跑

目标：Scene3 上层料盘 2D 框 + depth -> 3D JSON。

先启动 Scene3，然后另开终端保存 RGB：

```bash
cd ~/code/leju-kuavo-challenge-cup-2026-master/team_repo
python3 工具/视觉/save_compressed_image.py \
  --topic /cam_h/color/image_raw/compressed \
  --output vision_debug/scene3_seed3/scene3_head_rgb.jpg \
  --count 1
```

保存 depth：

```bash
python3 工具/视觉/save_depth_image.py \
  --topic /cam_h/depth/image_raw/compressedDepth \
  --output vision_debug/scene3_seed3/scene3_head_depth.npy \
  --png-output vision_debug/scene3_seed3/scene3_head_depth_vis.png \
  --count 1
```

检测料盘候选框：

```bash
python3 工具/视觉/detect_tray_opencv.py \
  --image vision_debug/scene3_seed3/scene3_head_rgb.jpg \
  --output vision_debug/scene3_seed3/scene3_detected.jpg
```

用检测结果里的 bbox 和 center 输出 3D JSON，例如：

```bash
python3 工具/视觉/detect_tray_3d.py \
  --rgb vision_debug/scene3_seed3/scene3_head_rgb.jpg \
  --depth vision_debug/scene3_seed3/scene3_head_depth.npy \
  --center 640,368 \
  --bbox 487,311,794,426 \
  --level upper \
  --output-json vision_debug/scene3_seed3/scene3_tray_result.json \
  --output-vis vision_debug/scene3_seed3/scene3_detected_3d.jpg
```

如果中文路径在服务器上不好复制，可以临时复制到英文目录：

```bash
mkdir -p tools/vision
cp 工具/视觉/*.py tools/vision/
```

然后把命令里的 `工具/视觉/` 换成 `tools/vision/`。
