# 窦欣悦：测试与提交路线

负责人：窦欣悦

你的任务不是写核心算法，而是保证大家做出来的东西：

```text
有记录
有证据
能复现
最后交得出去
```

## 你主要维护哪些文件

你主要看这 4 个位置：

```text
文档/测试记录.md
文档/提交检查清单.md
文档/团队分工.md
最终提交包/README.md
```

每天主要改：

```text
文档/测试记录.md
```

最后几天主要看：

```text
文档/提交检查清单.md
最终提交包/README.md
```

## 每天固定做什么

每天晚上找每个人要 5 个东西：

```text
1. 今天做了什么？
2. 现在能跑到哪一步？
3. 有没有截图、视频、输出文件或日志？
4. 卡在哪里？
5. 明天准备做什么？
```

然后把结果写进：

```text
文档/测试记录.md
```

## 你每天怎么填测试记录

打开：

```text
文档/测试记录.md
```

每个 scene 至少填一行：

| 日期 | Scene | Seed | 负责人 | 是否启动 | 是否识别 | 是否有 3D 点 | 是否动作执行 | 是否得分 | 失败原因 | 截图/视频 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 2026-07-13 | scene1 | 0 | 王仔俊 | 是/否 | 是/否 | 是/否 | 是/否 | 是/否 | 写清楚 | 路径或链接 |

不会判断就填：

```text
不确定，待确认
```

不要空着。

## 截图和视频怎么记

截图、小图片可以放 GitHub。

大视频、rosbag、大模型不要放 GitHub，写路径即可，例如：

```text
服务器路径：/home/shx/code/leju-kuavo-challenge-cup-2026-master/vision_debug/scene3_seed3/demo.mp4
网盘链接：xxx
```

## 每天晚上怎么问大家

可以直接复制这段发群里：

```text
请每个人按这个格式发我：

姓名：
负责 scene/模块：
今天做了什么：
现在能跑到哪一步：
截图/视频/输出文件路径：
卡点：
明天准备做什么：
```

收到后，你把信息整理进 `文档/测试记录.md`。

## 怎么判断一个结果有没有用

有用结果至少满足一个：

```text
有截图
有视频
有输出 JSON
有终端日志
有代码路径
有失败原因
```

如果只有一句“我调了一下”，没有证据，就记录为：

```text
无可验证结果
```

## 你不用做什么

你不用负责：

```text
写识别算法
写机械臂控制
调相机参数
训练模型
判断谁的方案最好
```

你只负责把事实记清楚。

## 最后提交前你要检查什么

打开：

```text
文档/提交检查清单.md
```

逐项确认：

```text
challenge_cup_task_template 是否在
challenge_task.py 是否在
README.md 是否在
三个 scene 是否都能启动
有没有 build/devel/log
有没有 rosbag 或大视频被打包进去
有没有写运行说明
有没有写依赖说明
有没有测试记录
```

## 最后要交的核心结构

最后不是直接交整个协作仓库，而是整理官方功能包：

```text
challenge_cup_task_template/
├── CMakeLists.txt
├── package.xml
├── README.md
└── scripts/
    └── challenge_task.py
```

如果大家新增了脚本，例如：

```text
scene1_task.py
scene2_task.py
scene3_task.py
vision_utils.py
```

也要放进 `challenge_cup_task_template`，并在 README 说明用途。

## 你每天的最小完成标准

每天结束前，你至少完成：

```text
1. 文档/测试记录.md 更新过
2. 每个 scene 至少有一条记录
3. 每个人的卡点写清楚
4. 截图/视频/输出文件路径写清楚
5. 明天要做什么写清楚
```

做到这些，你的任务就是合格的。
