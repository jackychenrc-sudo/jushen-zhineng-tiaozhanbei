# Day2 任务板

前提：服务器环境配置好，官方 Docker 能启动，三个 scene 能运行。

## Day2 总目标

不要追求满分。Day2 只追求把三个最小闭环跑起来：

```text
Scene1：能检测 / 接近 1 个包裹
Scene2：能识别 1 个零件并输出 3D 点
Scene3：能输出上层料盘 3D 点，动作骨架能动
```

## 统一启动命令

容器内：

```bash
cd /root/kuavo_ws
source devel/setup.zsh
rosrun challenge_cup_task_template challenge_task.py --scene scene3 --seed 3
```

换场景时改 `--scene scene1/scene2/scene3`。

## 闫帅辰：视觉与 Scene3 坐标

目标：固定 Scene3 seed3，输出上层料盘 3D 坐标。

步骤：

1. 保存头部 RGB。
2. 保存头部 depth。
3. 保存 `/cam_h/color/camera_info`。
4. 跑 `detect_tray_opencv.py` 得到 bbox 和 center_pixel。
5. 跑 `detect_tray_3d.py` 得到 JSON。
6. 把 JSON 发给王森做动作接入。

验收文件：

```text
vision_debug/scene3_seed3/scene3_head_rgb.jpg
vision_debug/scene3_seed3/scene3_head_depth.npy
vision_debug/scene3_seed3/scene3_camera_info.json
vision_debug/scene3_seed3/scene3_tray_result.json
vision_debug/scene3_seed3/scene3_detected_3d.jpg
```

## 王森：Scene3 动作骨架

目标：不等视觉完全准，先让动作链路能动。

步骤：

1. 在 `challenge_task.py --scene scene3` 分支里接入自己的动作函数。
2. 测试手臂轨迹接口。
3. 测试夹爪开合接口。
4. 写固定点动作：靠近上层区域、开夹爪、闭夹爪、回撤。
5. 等视觉 JSON 后，把固定点替换为视觉点。

Day2 验收：

```text
能从 scene3 入口触发一段固定动作，不要求成功抓出料盘。
```

## 王仔俊：Scene1 + 视觉辅助

目标：Scene1 先找 1 个包裹。

步骤：

1. 启动 Scene1。
2. 保存 RGB/depth/camera_info。
3. 先用颜色、边缘或轮廓找一个包裹候选框。
4. 输出包裹中心点和 3D 点。
5. 尝试让机械臂接近包裹。
6. 顺手帮 Scene3 多保存几组不同 seed 的 RGB/depth。

Day2 验收：

```text
有一个包裹中心点 JSON；有一次接近包裹的动作尝试。
```

## 史智涛 + Kevin：Scene2

目标：先识别 1 个零件，不做 6 个全流程。

步骤：

1. 启动 Scene2。
2. 保存 RGB/depth/camera_info。
3. 找到至少 1 个零件候选。
4. 判断类别：先粗分 A/B/C 任意一种也可以。
5. 输出类别、center_pixel、camera_xyz、目标箱。
6. 尝试接近或放入对应箱附近。

Day2 验收 JSON：

```json
{
  "scene": "scene2",
  "part_type": "a",
  "center_pixel": [0, 0],
  "depth": 0.0,
  "camera_xyz": [0.0, 0.0, 0.0],
  "target_bin": "purple"
}
```

## 窦欣悦：测试与提交

```text
测试表有记录；每个 scene 都有一条结果；至少有 3 张截图或 1 段视频路径。
```

## 晚上合并标准

能合并到 main 的内容必须满足：

- 文件路径清楚。
- README 或注释说明怎么运行。
- 不依赖本地绝对路径。
- 不提交大文件、rosbag、视频、build、devel、log。
- 至少有人在服务器上跑过一次。
