# 团队分工与交付物

## 总目标

一周内先做能得分、能解释、能提交的系统。不要把时间耗在复杂模型或大而全架构上。

每日检查三件事：

```text
今天有没有可运行代码？
今天有没有截图 / 视频 / 测试记录？
今天有没有比昨天多拿一点分？
```

## 工作边界

### 闫帅辰

只对视觉输出负责，不直接包办所有动作。

输出格式统一为：

```json
{
  "scene": "scene3",
  "object": "smt_tray",
  "level": "upper",
  "bbox": [487, 311, 794, 426],
  "center_pixel": [640, 368],
  "depth": 1.23,
  "camera_xyz": [0.0, 0.025, 1.23],
  "grasp_hint": "front_center"
}
```

### Scene1 组

先不要做 4 个包裹。先完成一个包裹闭环：

```text
看见一个包裹 -> 得到 3D 点 -> 机械臂接近 -> 抓取 -> 放到称重区或箱子附近
```

### Scene2 组

先不要做 6 个零件。先完成一个零件闭环：

```text
识别一个零件 -> 判断类别 -> 得到 3D 点 -> 放入对应箱子附近
```

### Scene3 组

先上层，后下层。

```text
上层料盘 10 分：优先稳定拿到
下层料盘 20 分：上层跑通后再尝试
```

### 测试与提交

每天晚上至少记录：

```text
scene | seed | 是否启动 | 是否识别 | 是否有 3D 点 | 是否动作执行 | 是否得分 | 失败原因 | 截图/视频路径
```
